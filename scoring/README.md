# Scoring
[draft in progress]

The score.py script calculates the similarity between a true driver-taken route and a predicted route generated by a user algorithm.

# Definition of Scoring Function:
The score awarded to the predicted route is calculated as follows:

$$score = \frac{SD(A,B) \cdot {ERP}_{norm}(A,B)}{{ERP}_e(A,B)}$$

where sequence $$A$$ is the historically realized sequence of deliveries, sequence $B$ is the algorithm-produced sequence of deliveries, $SD$ denotes the Sequence Deviation of $$B$$ with respect to $$A$$, $$ERP_{norm}$$ denotes the Edit Distance with Real Penalty applied to sequences $$A$$ and $$B$$ with normalized travel costs, and $${ERP}_e$$ denotes the number of edits done by the $$ERP$$ algorithm on sequence $$B$$ with respect to $$A$$.

If the user-submitted route is deemed invalid, meaning it does not begin at the station and contain all stops, the scoring function returns a score equivalent to a typical complete random shuffle of the driver-taken sequence.

# _Evaluate_ Function
The 'evaluate' function applies the scoring logic defined above to a single route or set of routes for which the inputs below are defined.

## Inputs:
The 'evaluate' function requires 4 inputs:
- actual_routes_json: the filepath of the JSON object containing the actual driver-taken route(s), given as a string
- submission_json: the filepath of the JSON object containing the user-submitted route(s), given as a string
- cost_matrices_json: the filepath of the JSON object containing the estimated transit times between all possible stops in the route(s), given as a string
- invalid_scores_json: the filepath of the JSON object containing the scores assigned to user-submitted route(s) deemed invalid, given as a string


## Outputs:
The 'evaluate' function returns 1 output:
- scores_json: a JSON object providing the scores awarded to all user-submitted routes
